# Contributing

Thanks for helping improve effect-spacetimedb. Issues, bug reports, questions,
and suggestions are welcome.

This project does not accept external pull requests. Please open an issue with
the smallest reproduction or proposal you can provide. Maintainers review
reports, make the code changes, and credit reporters in the relevant release
notes when their report leads to a fix or improvement.

Releases are cut by the maintainers. If you need a fix on a particular timeline,
include that context in the issue so maintainers can weigh it against other
work.

## Useful Reports

Good bug reports usually include:

- The SpaceTimeDB and effect-spacetimedb versions involved.
- A minimal module, reducer, subscription, or client snippet that reproduces the
  behavior.
- The expected behavior and the actual behavior.
- Any relevant compiler, runtime, or generated-client output.

For security-sensitive reports, follow `SECURITY.md` instead of opening a public
issue.
