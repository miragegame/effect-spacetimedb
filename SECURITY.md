# Security Policy

Please report suspected vulnerabilities privately instead of opening a public
issue.

Use GitHub's private vulnerability reporting from the repository's Security tab,
then send a concise report with:

- A description of the vulnerability and likely impact.
- Steps to reproduce, proof-of-concept code, or affected APIs when available.
- Any known mitigations or workarounds.

Maintainers will acknowledge credible reports, investigate them, and coordinate
fixes and releases. Public credit is welcome when disclosure is appropriate, but
reporters may also request anonymity.
